## [CRITICAL] 시점 기반 컬럼 이해 (작성 전 필수!)

> **핵심 원리**: FMEA 컬럼은 시점(과거/현재/미래)으로 구분됨
> 이 개념을 이해하면 고장형태/고장영향/고장원인 혼동 방지

### 시간 흐름

```
[과거]           [과정]           [현재]           [미래]
F열 (원인)  -->  G열 (메커니즘)  -->  E열 (형태)  -->  C열 (영향)
용접 불량   -->  응력집중        -->  크랙 발생   -->  지락사고
설계 오류   -->  피로           -->  파손        -->  변압기 트립
재료 불량   -->  크리프          -->  변형        -->  과열
```

### 컬럼별 정의

| 시점 | 컬럼 | 핵심 질문 | 정의 | 예시 |
|------|------|----------|------|------|
| **과거** | F열 (고장원인) | "왜 발생했나?" | 고장의 근본 원인 | 설계 오류, 재료 불량, 용접 불량 |
| **과정** | G열 (고장메커니즘) | "어떤 과정으로?" | 원인->형태 사이의 물리적/화학적 과정 | 피로, 응력집중, 크리프, 열화진행 |
| **현재** | E열 (고장형태) | "지금 눈에 보이나?" | 현재 시각적으로 관찰 가능한 현상 | 크랙, 변형, 이완, 부식, 파손 |
| **미래** | C열 (고장영향) | "앞으로 무슨 일?" | 기능 실패로 인해 미래에 발생하는 결과 | 지락사고, 과열, 변압기 트립 |

## [CRITICAL] E열 작성 전 판단 질문

> **작성하려는 현상에 대해 아래 질문을 순서대로 묻기**

### 판단 흐름도

```
Q1: "이것을 보려면 측정 장비가 필요한가?"
    |
    +-- YES --> G열 (고장메커니즘) 또는 측정값
    |           예: 와전류 증가, 철손 증가, 저항 증가
    |
    +-- NO --> Q2로 진행

Q2: "이것이 지금 당장 육안/간단한 검사로 보이는가?"
    |
    +-- YES --> E열 (고장형태) [O] 올바름
    |           예: 크랙, 변형, 부식, 이완, 파손
    |
    +-- NO --> Q3로 진행

Q3: "이것은 운전 후에야 알 수 있는 결과인가?"
    |
    +-- YES --> C열 (고장영향)
    |           예: 소음, 진동, 트립, 지락사고, 과열
    |
    +-- NO --> Q4로 진행

Q4: "이것은 과거에 잘못된 원인인가?"
    |
    +-- YES --> F열 (고장원인)
    |           예: 설계 오류, 재료 불량, 용접 불량
    |
    +-- NO --> Q5로 진행

Q5: "이것은 원인과 형태 사이의 과정인가?"
    |
    +-- YES --> G열 (고장메커니즘)
    |           예: 피로, 응력집중, 크리프, 열화진행
    |
    +-- NO --> 분류 재검토 필요
```

### 자가 검증 질문 (작성 후)

| 컬럼 | 검증 질문 | YES면 |
|------|----------|-------|
| **E열 (형태)** | "이것이 지금 당장 눈에 보이는가?" | 올바름 |
| **C열 (영향)** | "이것이 앞으로 고객에게 영향을 주는가?" | 올바름 |
| **F열 (원인)** | "이것이 과거에 잘못된 것인가?" | 올바름 |
| **G열 (메커니즘)** | "이것은 눈에 안 보이는 과정인가?" | 올바름 |

## SECTION:FORBIDDEN_MECHANISM_TERMS

> **[BLOCKING]** E열(고장형태)에 메커니즘 용어 금지!
> **이유**: 메커니즘은 "과정"이며, 눈에 보이지 않음. 분석/측정해야 알 수 있음
> **조치**: 메커니즘은 G열(고장메커니즘)에 작성

### 금지 메커니즘 용어 (G열로 이동!)

메커니즘: 피로, 응력집중, 크리프, 열화진행, 부식진행, 산화진행, 마모진행, 열피로, 기계피로, 열팽창, 열수축, 와전류, 히스테리시스

### 메커니즘 vs 형태 구분

> **핵심**: 메커니즘 = "과정" (눈에 안 보임) / 형태 = "결과" (눈에 보임)

| 용어 | 눈에 보이나? | 올바른 위치 | 설명 |
|------|------------|-----------|------|
| 피로 | X (과정) | G열 | 반복 하중에 의한 손상 축적 과정 |
| 응력집중 | X (과정) | G열 | 특정 부위에 응력이 집중되는 과정 |
| 크리프 | X (과정) | G열 | 장시간 하중에 의한 변형 진행 과정 |
| 열화진행 | X (과정) | G열 | 재료 성질이 저하되는 과정 |
| 크랙 | O (보임) | E열 | 표면에 보이는 균열 |
| 파손 | O (보임) | E열 | 부품이 깨진 상태 |
| 변형 | O (보임) | E열 | 형상이 변한 상태 |
| 이완 | O (보임) | E열 | 체결이 풀린 상태 |

### 잘못된 작성 -> 올바른 작성

| 잘못된 E열 | 문제점 | 올바른 E열 | 올바른 G열 |
|-----------|--------|-----------|-----------|
| 피로 파손 | "피로"는 과정, 안 보임 | 파손 | 피로 |
| 응력집중 | 과정, 안 보임 | 크랙, 균열 | 응력집중 |
| 크리프 변형 | "크리프"는 과정 | 변형 | 크리프 |
| 열피로 크랙 | "열피로"는 과정 | 크랙 | 열피로 |
| 부식진행 | "진행"은 과정 | 부식 | 부식진행 |
| 열화 | 추상적, 안 보임 | 변색, 균열, 탄화 | 열화진행 |

## SECTION:FORBIDDEN_MEASUREMENT_VALUES

> **[BLOCKING]** E열(고장형태)에 측정값 금지!
> **이유**: 측정해야 알 수 있는 값은 "눈에 보이는 현상"이 아님
> **조치**: 측정값은 G열(고장메커니즘)에 작성

### 금지 패턴: ~증가, ~저하, ~상승, ~감소, ~초과, ~오차

| 금지 유형 | 금지 용어 | 이유 | 올바른 대안 |
|----------|----------|------|------------|
| **측정 증가** | 와전류 증가, 철손 증가, 손실 증가, 저항 증가 | 측정해야 알 수 있음 | G열로 이동, E열은 "변색", "과열흔적" |
| **측정 저하** | 효율 저하, 성능 저하, 절연 저하, 강도 저하 | 측정해야 알 수 있음 | G열로 이동, E열은 구체적 현상 |
| **측정 상승** | 온도 상승, 압력 상승, 전압 상승 | 측정해야 알 수 있음 | E열은 "과열흔적", "변색" |
| **측정 오차** | 조립오차, 정렬오차, 치수오차 | 측정해야 알 수 있음 (mm) | E열은 "정렬이탈", "위치편심", "간격과대" |

## SECTION:TAG_REQUIREMENTS

> **[BLOCKING]** 고장형태 태그 필수 (부족:/과도:/유해:)

### 필수 태그 형식

```
형식: [부족|과도|유해]: [관찰 가능한 현상]
예시: 부족: 이완, 과도: 과압착흔, 유해: 와전류발열흔
```

### 태그 선택 기준

> **핵심 개념**: 태그는 "기능 결과물"에 적용!
> 기능 자체가 아니라, **기능이 만들어내는 결과물**이 부족/과도/유해한지 판단

| 태그 | 질문 | 의미 | 비유 |
|------|------|------|------|
| **부족:** | "기능 결과물이 목표에 못 미치나?" | 기능 미달 | 수도관에서 물이 너무 적게 나옴 |
| **과도:** | "기능 결과물이 목표를 초과하나?" | 기능 과잉 | 수도관에서 물이 너무 세게 나옴 |
| **유해:** | "기능은 정상인데 부작용이 있나?" | 부작용 발생 | 물은 잘 나오는데 옆에서 샘 |

### 태그 선택 플로우차트

```
Q1. 기능 결과물이 목표에 못 미치나?
    |
    +-- YES --> 부족:
    +-- NO --> Q2로

Q2. 기능 결과물이 목표를 초과하나?
    |
    +-- YES --> 과도:
    +-- NO --> Q3로

Q3. 기능은 정상인데 부작용이 있나?
    |
    +-- YES --> 유해:
    +-- NO --> 해당 없음 (기능 정상)
```

### 태그 판단 2단계 프로세스

**1단계: 기능 결과물 식별**

```
기능: "자속을 전달한다"
  -> 결과물: "전달된 자속" (자속 밀도)

기능: "클램핑 압력을 인가한다"
  -> 결과물: "인가된 압력" (압력값)

기능: "절연을 유지한다"
  -> 결과물: "유지된 절연" (절연 성능)
```

**2단계: 결과물이 부족/과도/유해한지 판단**

```
기능: "자속을 전달한다" -> 결과물: "전달된 자속"
  - 부족: 전달된 자속이 부족하면? -> 적층갭, 자속밀도 저하
  - 과도: 전달된 자속이 과도하면? -> 자기포화, 과열
  - 유해: 자속 전달은 정상인데 부작용? -> 와전류 발생
```

### 부품별 태그 적용 예시

**철심 (CORE) - "자속을 전달한다"**

| 태그 | 질문 | 결과물 상태 | 고장형태 |
|------|------|-----------|---------|
| 부족: | 전달이 덜 되면? | 자속이 적게 전달됨 | 부족: 적층이완 |
| 과도: | 전달이 너무 많으면? | 자속이 과하게 전달됨 | 과도: 자기포화 변색 |
| 유해: | 전달은 되는데 딴 데서? | 자속은 정상, 부작용 발생 | 유해: 와전류 발열흔 |

**클램프 (CLAMP) - "클램핑 압력을 인가한다"**

| 태그 | 질문 | 결과물 상태 | 고장형태 |
|------|------|-----------|---------|
| 부족: | 압력이 덜 가해지면? | 클램핑 압력 부족 | 부족: 볼트 이완, 간극 발생 |
| 과도: | 압력이 너무 세면? | 클램핑 압력 과다 | 과도: 절연물 파손, 권선 변형 |
| 유해: | 압력은 적정한데 딴 데서? | 압력 정상, 부작용 발생 | 유해: 국부 응력 크랙 |

### "과도" N/A 처리 가이드

> 모든 기능에 3가지 태그가 다 적용되는 것은 아닙니다.

| 기능 | 과도가 문제되나? | 처리 |
|------|----------------|------|
| 절연을 유지한다 | X (절연이 너무 좋아서 문제 없음) | 과도: N/A |
| 접지 경로를 제공한다 | X (접지가 너무 잘 되어서 문제 없음) | 과도: N/A |
| 기밀을 유지한다 | X (기밀이 너무 좋아서 문제 없음) | 과도: N/A |

**주의**: N/A인 경우 해당 태그는 작성하지 않습니다. 무리하게 만들지 마세요!

### 잘못된 태그 판단 예시

| 기능 | 고장형태 | 잘못된 태그 | 올바른 태그 | 이유 |
|------|---------|------------|------------|------|
| 자속을 전달한다 | 와전류 발열흔 | 부족: | **유해:** | 자속 전달은 정상, 부작용으로 와전류 발생 |
| 클램핑 압력을 인가한다 | 절연물 파손 | 유해: | **과도:** | 압력이 너무 세서 파손 (부작용 아님) |
| 진동을 억제한다 | 체결 이완 | 과도: | **부족:** | 억제가 덜 되어 진동->이완 (억제 부족) |
| 절연을 유지한다 | 절연물 수축 | 부족: | **유해:** | 절연은 정상, 열수축 부작용 발생 |
| 손실을 최소화한다 | 자기포화흔 | 과도: | - | [X] 자기포화는 **자속 과도**의 결과, 손실 최소화와 무관! |
