## 3. S/O/D 값 자동 산정 알고리즘

### 3.1 S값 산정 (심각도)

**기본 산정: 중요_경미 + 치명도**

| 중요_경미 (Col 14) | 치명도 (Col 15) | S값 | 건수 |
|-------------------|----------------|-----|------|
| 중요 | 상 | **9-10** | 68 |
| 중요 | 중 | **7-8** | 327 |
| 경미 | 중 | **5-6** | 631 |
| 경미 | 하 | **3-4** | 539 |
| 분류제외 | - | 2 | 204 |

**[신규] 가중치 적용**:

1. **분류 가중** (Col 1):
   - CLAIM (외부 발견): S값 +1
   - NCR (내부 발견): S값 유지

2. **피해보상비 가중** (Col 34):
   - 1억원 이상: S=10 (강제)
   - 1천만원 이상: S값 +1 (최대 10)
   - 1천만원 미만: S값 유지

**S값 최종 공식**:
```python
def calc_s_value(row):
    # 기본값: 중요_경미 + 치명도
    base = get_base_s(row['중요_경미'], row['치명도'])

    # 분류 가중 (CLAIM +1)
    if row['분류'] == 'CLAIM':
        base += 1

    # 피해보상비 가중
    cost = row['피해보상비'] or 0
    if cost >= 100_000_000:  # 1억 이상
        return 10
    elif cost >= 10_000_000:  # 1천만 이상
        base += 1

    return min(base, 10)
```

### 3.2 O값 산정 (발생빈도)

**기본 산정: 12년간 빈도**

| 12년간 발생 건수 | O값 |
|-----------------|-----|
| 50건 이상 | 8 |
| 20-49건 | 6 |
| 10-19건 | 4 |
| 5-9건 | 3 |
| 1-4건 | 2 |

**[신규] 발생년도 트렌드 가중** (Col 3):

최근 3년 (2024-2026) 가중치 x1.5 적용:
```python
def calc_o_value(part_name):
    # 전체 건수
    total = count_by_part(part_name)

    # 최근 3년 건수
    recent = count_by_part_recent_3y(part_name)

    # 가중치 적용 (최근 비중 높으면 O값 상향)
    recent_ratio = recent / total if total > 0 else 0

    base_o = get_base_o(total)

    if recent_ratio > 0.5:  # 최근 3년이 50% 이상
        return min(base_o + 1, 10)
    elif recent_ratio < 0.2:  # 최근 3년이 20% 미만 (개선 추세)
        return max(base_o - 1, 1)

    return base_o
```

```python
def calc_d_value(row):
    base = {
        '수입': 3,
        '공정': 5,
        '제품': 7,
        '시공': 9
    }.get(row['검사구분'], 5)

    # 항목구분 보정
    if '절연시험' in str(row['항목구분']):
        base -= 1
    elif '완성' in str(row['항목구분']):
        base += 1

    return min(max(base, 1), 10)
```

## 4. SQL 쿼리 템플릿

### 4.1 부품별 전체 분석 (확장)

```sql
SELECT
    품명,
    COUNT(*) as 총건수,
    -- 분류별
    SUM(CASE WHEN 분류='CLAIM' THEN 1 ELSE 0 END) as CLAIM건수,
    SUM(CASE WHEN 분류='NCR' THEN 1 ELSE 0 END) as NCR건수,
    -- S값 관련
    SUM(CASE WHEN 중요_경미='중요' THEN 1 ELSE 0 END) as 중요건수,
    MAX(피해보상비) as 최대피해보상비,
    -- 최근 트렌드
    SUM(CASE WHEN 발생년도 >= 2024 THEN 1 ELSE 0 END) as 최근3년건수,
    -- 검출 시점
    GROUP_CONCAT(DISTINCT 검사구분) as 검출시점,
    -- 원인 분류
    GROUP_CONCAT(DISTINCT 발생원인유형) as 원인유형,
    GROUP_CONCAT(DISTINCT 원인부서) as 원인부서
FROM qa_records
WHERE 품명 LIKE '%{부품명}%'
GROUP BY 품명
ORDER BY 총건수 DESC;
```

### 4.2 E열 (고장형태) 추출

```sql
SELECT

```

### 4.3 F열 (고장원인) + 자동 태그

```sql
SELECT
    발생원인유형,
    원인부서,
    발생원인,
    COUNT(*) as 건수,
    CASE
        WHEN 발생원인유형 IN ('부품 불량', '자재 불량') THEN '재료:'
        WHEN 발생원인유형 LIKE '%생산%' THEN '제작:'
        WHEN 발생원인유형 LIKE '%설계%' THEN '설계:'
        WHEN 발생원인유형 LIKE '%검사%' THEN '시험:'
        WHEN 원인부서 = '구매' THEN '재료:'
        WHEN 원인부서 IN ('생산', '시공') THEN '제작:'
        WHEN 원인부서 LIKE '%설계%' THEN '설계:'
        WHEN 원인부서 = 'QA' THEN '시험:'
        ELSE '기타:'
    END as FMEA태그
FROM qa_records
WHERE 품명 LIKE '%{부품명}%'
  AND 발생원인 IS NOT NULL
  AND 발생원인 != '-'
GROUP BY 발생원인유형, 원인부서, 발생원인
ORDER BY 건수 DESC
LIMIT 10;
```

### 4.4 S/O/D 자동 산정

```sql
SELECT
    품명,
    발생현상유형소분류 as 고장형태,
    COUNT(*) as 빈도,
    -- S값 산정
    MAX(CASE
        WHEN 피해보상비 >= 100000000 THEN 10
        WHEN 중요_경미='중요' AND 치명도='상' THEN 9
        WHEN 중요_경미='중요' AND 치명도='중' THEN 7
        WHEN 중요_경미='경미' AND 치명도='중' THEN 5
        ELSE 3
    END) + MAX(CASE WHEN 분류='CLAIM' THEN 1 ELSE 0 END) as S값,
    -- O값 산정
    CASE
        WHEN COUNT(*) >= 50 THEN 8
        WHEN COUNT(*) >= 20 THEN 6
        WHEN COUNT(*) >= 10 THEN 4
        WHEN COUNT(*) >= 5 THEN 3
        ELSE 2
    END as O값기본,
    -- D값 산정
    MIN(CASE
        WHEN 검사구분='수입' THEN 3
        WHEN 검사구분='공정' THEN 5
        WHEN 검사구분='제품' THEN 7
        ELSE 9
    END) as D값
FROM qa_records
WHERE 품명 LIKE '%{부품명}%'
GROUP BY 품명, 발생현상유형소분류
ORDER BY 빈도 DESC;
```

### 4.5 비용 기반 우선순위 (RPN 검증)

```sql
SELECT
    품명,
    발생현상유형소분류 as 고장형태,
    COUNT(*) as 빈도,
    SUM(발생_비용_합계) as 총비용,
    MAX(피해보상비) as 최대피해보상,
    AVG(발생_비용_합계) as 평균비용
FROM qa_records
WHERE 품명 LIKE '%{부품명}%'
  AND 발생_비용_합계 > 0
GROUP BY 품명, 발생현상유형소분류
ORDER BY 총비용 DESC
LIMIT 10;
```
