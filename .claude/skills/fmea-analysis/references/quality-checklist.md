# FMEA 품질 체크리스트

**목적**: FMEA 완료 전 최종 품질 검증
**사용 시점**: FMEA 작성 완료 후, Excel 생성 전

---

## 🔴 MUST (필수 - 90점 달성)

### 1. 데이터 소스 검증
**내부 문서** (있는 경우 우선):
- [ ] 도면 (PDF) 확인 → 치수, 재질, 공차 추출
- [ ] 작업기준서 확인 → 공정 조건, 검사 기준 추출
- [ ] CHECK SHEET 확인 → 검사 항목, 측정 방법 추출
- [ ] 내부 문서 정보가 FMEA 항목에 실제 반영됨

**WebSearch** (5개 출처 필수):
- [ ] IEC/IEEE/CIGRE/Transformer Magazine/일반 모두 검색
- [ ] 표준/문헌 기반 항목만 사용
- [ ] 출처 Row 3에 실제 사용한 출처만 명시 (내부문서 + WebSearch)

### 2. 다이아몬드 확장 구조 [*]
- [ ] Failure Chain 논리적 연결 (기능→형태→영향, 원인은 형태에서 도출)
- [ ] WebSearch에서 발견된 항목만 사용 (억지 확장 금지)
- [ ] 같은 형태에 여러 원인 = 정상 (다이아몬드 구조)

### 3. 기능 분석
- [ ] WebSearch에서 발견된 주요 기능 사용
- [ ] 억지로 기능 개수를 맞추지 않음

### 4. 고장메커니즘
- [ ] 원인 → 과정 → 결과 논리적 연결
- [ ] WebSearch 기반 실제 메커니즘 사용

### 5. S/O/D 값
- [ ] S 값 비용/손실/복구 기간 명시
- [ ] O 값 WebSearch 통계 근거 (CIGRE/IEEE)
- [ ] D 값 검출률 추정

### 6. 대책 차별화
- [ ] 각 항목마다 구체적이고 다른 대책
- [ ] WebSearch에서 발견된 실제 대책 사용 (억지 생성 금지)

### 7. Python 스크립트 사용
- [ ] **반드시** `generate_fmea_excel.py` 사용
- [ ] 직접 코딩 금지 (셀 병합 오류 원인)

### 8. [LOCK] 병합 연속성 (FMEA Facilitator) [*] NEW
**목적**: Excel 셀 병합이 가능하도록 동일 값 연속 배치 보장

**기능별 연속성**:
- [ ] 동일 기능 항목이 연속 배치됨 (비연속 시 B열 병합 불가!)
- [ ] 기능별 완결 후 다음 기능으로 이동 (기능A 완료 → 기능B 시작)

**고장영향별 연속성**:
- [ ] 동일 고장영향 항목이 연속 배치됨 (비연속 시 C열 병합 불가!)

**검증 방법**:
```
기능별 항목 위치 확인:
- [기능1] Row 7-12 [OK] 연속
- [기능2] Row 13-20 [OK] 연속
- [기능3] Row 21-25, 30-32 [X] 비연속 → 수정필요!
```

**[!] 비연속 발견 시**: 해당 기능 항목을 연속 배치로 재정렬

### 9. 용어 통일 (FMEA Facilitator) [*] NEW
**목적**: 일관된 표현으로 문서 품질 향상

**금지 용어 → 권장 표현**:
| 금지 | 권장 | 이유 |
|------|------|------|
| 불량 | 부족, 오류, 손상, 누락 | 추상적 → 구체적 |
| 과도 | 초과, 과잉, 과부하 | 추상적 → 정량화 가능 |
| 부적절 | 미달, 불일치, 편차 | 추상적 → 측정 가능 |
| 미흡 | 부족, 미달, 결핍 | 추상적 → 구체적 |

**고장형태 표현**:
- [ ] 구체적 현상 사용: 층간단락, 과열, 변형, 부식, 균열, 누유
- [ ] 추상적 표현 금지: "저하", "불량", "문제"

**문서 정합성**:
- [ ] 맞춤법 검증 (조사 "을/를", "이/가" 정확)
- [ ] 문맥 일관성 (주어-술어 호응)
- [ ] 동일 현상 → 동일 표현 (일관성)
