### 1. [!] CRITICAL: 체계적 준비의 중요성

**현대자동차의 20년 시행착오 교훈에서 배운 핵심 원칙**:
- **급하게 작성 시작 금지**: 충분한 준비 없이 FMEA 작성하면 품질 저하 및 재작업 발생
- **체계적 준비 필수**: 부품 분석, 과거 문제 검토, 노이즈 인자 파악 후 작성 시작
- **과거 문제 DB 우선 활용**: 유사 부품의 과거 문제 60% 재활용 가능

**절대 원칙** (CRITICAL):
```
[OK] 올바른 순서 (방법 1):
준비 및 분석 → FMEA 작성 → 검증 및 완료

[X] 잘못된 순서:
바로 Excel 작성 시작 → 중간에 정보 수집 → 재작업 반복!
```

**실무 주의사항**:
- **Excel 파일 생성 전에 준비 완료** (부품 분석, WebSearch 기반 기술 정보 수집)
- **과거 문제 DB 우선 조회** (60% 재활용 목표)
- **5대 노이즈 인자 사전 검토** (작성 중 추가하면 누락 가능성)
- 현대차는 이와 유사한 문제로 20년간 재작업 반복 → 체계적 준비의 중요성 입증

### 2. 방법 1 체계적 준비 워크플로우 (3단계)

#### **1단계: 준비 및 분석 (Preparation & Analysis)**

**목표**: 충분한 정보 수집 후 FMEA 작성 시작

[OK] **필수 수집 자료**:
- BOM (Bill of Materials)
- 설계 도면 (CAD, PDF)
- 과거 품질 이슈 기록
- 과거 문제 DB (`04.HMS_DB/QA자료/01_원본데이터/QA_품질이력.db`)
- 변경 사항 (신규 개발, 설계 변경, 재료 변경)

[OK] **WebSearch 기반 기술 정보 수집**:
- IEC/IEEE/CIGRE/Transformer Magazine 5개 출처 검색
- 실제 고장 형태 및 원인 추출
- 자연스럽게 항목 수 결정 (품질 우선, 개수 제한 없음)
- 주요 기능 도출 (1-4개)
- 주변 시스템/부품 파악 (Boundary)

[OK] **5대 노이즈 인자 검토**:
- 1. 외부 환경 (온도, 습도, 낙뢰, 염해)
- 2. 고객 사용 (과부하, 단락, 전압 변동)
- 3. 시스템 상호작용 (고조파, 공진)
- 4. 시간 경과 (절연 열화, 기계적 마모)
- 5. 부품 간 변동 (제작 공차, 재료 특성 편차)

[OK] **과거 문제 DB 조회** (최우선):
- `04.HMS_DB/QA자료/01_원본데이터/QA_품질이력.db` 조회 (SQLite)
- 유사 고장 형태 검색
- 원인/메커니즘 재활용 (60% 목표)
- 대책 및 검증 방법 참조

**1단계 완료 기준**:
- [ ] WebSearch 5개 출처 검색 완료 (IEC/IEEE/CIGRE/Transformer Magazine/일반)
- [ ] 실제 고장 형태/원인 추출 완료
- [ ] 주요 기능 리스트 작성 (1-4개)
- [ ] 과거 문제 DB 조회 완료
- [ ] 5대 노이즈 인자 체크리스트 작성

#### **2단계: FMEA 작성 (Creation)**

**목표**: 1단계 분석 결과를 바탕으로 체계적 FMEA 작성

[OK] **고장 형태 도출** (구체적 현상으로 작성):
- **드러나는 형태로 기술**: 변형, 크랙, 부식, 과열, 단락, 누유, 소음, 진동 등
- **부품별 고장형태**: 층간단락, 절연열화, 균열, 이완, 탈락, 피로파손 등
- [X] **추상적 표현 금지**: "부족", "저하", "과도" 등
- **참조**: [failure-mode-diversity.md](failure-mode-diversity.md) 부품별 목록

[OK] **고장 원인 분석** (제품 라이프사이클 7단계, FMEA는 1-4단계 주로 사용):
- 설계 (설계불량): 사양 오류, 계산 오류, 공차 문제
- 재료 (재료불량): 재질 결함, 오염, 특성 편차
- 제작 (제작불량): 조립 오류, 용접 불량, 오염
- 시험 (시험불량): 시험 절차 오류, 측정 오류, 교정 문제
- 운송 (운송불량): 진동 손상, 충격 손상, 환경 노출
- 설치 (설치불량): 설치 오류, 연결 오류, 정렬 문제
- 운영 (운영불량): 운전 조건 위반, 유지보수 오류, 과부하

[OK] **S/O/D/AP 평가** (변압기 특화 기준):
- S (Severity): 고장 영향의 심각도 (1-10, 변압기 기준)
- O (Occurrence): 발생 가능성 (1-10, 연간 생산 50대 기준)
- D (Detection): 검출 난이도 (1-10, 변압기 검출 방법)
- AP (Action Priority): H/M/L (S/O/D 조합 기반)

[OK] **Excel 파일 작성**:
- 20개 컬럼 구조 (방법 1 표준)
- Row 1: 제목 "{부품명}_FMEA" (굵게, 16pt, 가운데)
- Row 2: "프로젝트: 변압기"
- Row 3: "자료 출처: [실제 사용한 출처만]"
- Row 4: 빈 행 (구분선)
- Row 5: AIAG-VDA 7-Step 프로세스 구분 (녹색 `70AD47`)
- Row 6: 헤더 (파란색 `4472C4`)
- Row 7+: 데이터
- 셀 병합 적용 (컬럼 1-5: 부품명, 기능, 고장 영향, S, 고장 형태)
- 모든 셀에 테두리 적용
- A3 가로 방향, 1페이지에 모든 열 맞춤

**2단계 완료 기준**:
- [ ] WebSearch 기반 항목 생성 완료 (품질 우선, 개수 제한 없음)
- [ ] 구체적 고장형태 사용 확인 (변형, 크랙, 과열 등)
- [ ] 4단계 라이프사이클 반영 확인 (설계/재료/제작/시험)
- [ ] S/O/D/AP 평가 완료 (변압기 기준)
- [ ] Excel 양식 표준 준수

### 3. 핵심 원칙 (재강조)

**[!] 준비 없이 작성 시작하지 말 것**
- 급하게 작성 → 품질 저하 → 재작업 반복
- 충분한 준비 (1단계) 후 작성 시작 (2단계)

**[!] 과거 문제 DB는 핵심 자산**
- 60% 재활용 목표
- 유사 부품의 과거 문제 우선 조회
- 원인/메커니즘/대책 재활용

**[!] 체계적 분석 후 작성**
- WebSearch 기반 기술 정보 수집 → 자연스럽게 항목 수 결정
- 5대 노이즈 인자 사전 검토
- 7단계 라이프사이클 반영
