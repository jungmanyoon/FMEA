## G열 (고장메커니즘) - 화살표 체인 필수!

**형식**: `원인 -> 과정 -> 결과` (3단계 화살표 체인)

| 올바른 예시 | 잘못된 예시 |
|------------|------------|
| `용접 불량 -> 응력 집중 -> 피로 크랙` | `피로` (단어만) |
| `체결 부족 -> 이완 진행 -> 유격 발생` | `응력집중` (과정만) |
| `온도 상승 -> 절연 열화 -> 부분방전` | `크리프 변형` (결과만) |

**검증**: validate_mechanism() - 화살표(->) 2개 이상 필수

---

## H열 (현재예방대책) - 멀티라인 + 기준값 필수!

**형식**: 4-8줄, 각 단계별 대책 + 출처 + 기준값

```
설계: [대책1] (출처)
- [세부항목] [기준값] (출처)
재료: [대책2] (출처)
- [세부항목] [기준값] (출처)
제작: [대책3] (출처)
- [세부항목] [기준값] (출처)
시험: [대책4] (출처)
- [세부항목] [기준값] (출처)
```

**예시** (올바름):
```
설계: 클램프 강도 설계 검토 (중신 CHECK SHEET 설계-No.12)
- 허용응력 계산: 안전율 2.0 이상 (IEQT-T-C018 SS3.2)
재료: 클램프 자재 수입검사 (IEQT-T-R018 SS4.1)
- 재질증명서 확인: SS400 이상 (수입검사 기준)
제작: 용접부 육안검사 (IEQT-T-W002 SS5.3)
- 용접 비드 외관: 언더컷 없음 (IEQT-T-C018 SS6.2)
시험: UT 검사 실시 (IEQT-T-W002 SS7.1)
- 결함 허용기준: 2mm 이하 (KS B 0896)
```

**잘못된 예시** (BLOCKING):
```
설계: 설계 검토 (일반)   <- 1줄만, 기준값 없음!
```

## J열 (현재검출대책) - 멀티라인 + 합격기준 필수!

**형식**: H열과 동일 (4-8줄, 단계별 검출방법 + 합격기준)

**예시** (올바름):
```
설계: 도면 승인 (설계팀)
- 클램프 강도 검토: 계산서 첨부 필수 (설계 프로세스)
재료: 수입검사 성적서 (QA팀)
- 재질증명서: SS400 이상 확인 (수입검사 기준)
제작: 용접 후 육안검사 (생산팀)
- 비드 외관: 크랙/언더컷 없음 (IEQT-T-W002 SS5.3)
시험: UT 검사 (QA팀)
- 결함 크기: 2mm 이하 (KS B 0896)
```
