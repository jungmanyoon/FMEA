## 7-Step 개요

| Step | 단계명 | 목적 | 주요 활동 |
|------|--------|------|-----------|
| 1 | Planning and Preparation | 계획 수립 | 범위, 팀 구성, 일정 |
| 2 | Structure Analysis | 구조 분석 | 시스템 분해, 블록 다이어그램 |
| 3 | Function Analysis | 기능 분석 | 기능 정의, 요구사항 |
| 4 | Failure Analysis | 고장 분석 | 고장 형태, 원인, 영향 |
| 5 | Risk Analysis | 위험 분석 | S/O/D 평가, AP 계산 |
| 6 | Optimization | 최적화 | 대책 수립, 개선 |
| 7 | Results Documentation | 결과 문서화 | 보고서, 추적 |

## Step 1: 계획 및 준비 (Planning and Preparation)

### 목적
FMEA 범위 및 팀 구성

### 수행 작업
1. **범위 정의**
   - 분석 대상: 154kV~765kV 변압기
   - 부품 선정: CORE, WINDING, BUSHING 등
   - 분석 깊이: 부품 단위 (하위 구성요소 포함)

2. **팀 구성**
   - DFMEA: 전기설계팀, 구조설계팀, 개발팀
   - PFMEA: 생산팀, 생산기획팀, QA팀
   - Facilitator: FMEA 전문가

3. **일정 계획**
   - 킥오프 미팅
   - 정기 검토 회의
   - 최종 보고

4. **사전 조사**
   - 생산/폐기/환경 조사
   - 과거 문제점 및 변경점 조사
   - 일정 및 담당자 지정

**출력물**: 프로젝트 계획서, 팀 구성표

---

## Step 2: 구조 분석 (Structure Analysis)

### 목적
시스템 계층 구조 파악 (BOM 분석)

### 수행 작업
1. **시스템 분해**
   ```
   변압기 (System)
   ├─ CORE (철심)
   ├─ WINDING (권선)
   │   ├─ 1차 코일
   │   └─ 2차 코일
   ├─ TANK (탱크)
   ├─ BUSHING (부싱)
   ├─ TAP_CHANGER (탭 절환기)
   └─ COOLING_SYSTEM (냉각 시스템)
   ```

2. **블록 다이어그램 작성**
   - 부품 간 연결
   - 입출력 관계
   - 인터페이스

3. **Focus Element 선정** (분석 대상 부품/시스템)

**출력물**: BOM, 구조도, Focus Element 정의서

## Step 3: 기능 분석 (Function Analysis)

### [!] CRITICAL: Step 3-4-5는 연속 작성!

### 목적
각 부품/시스템의 기능 정의

### 수행 작업
1. **사전 조사**: 가공/조립/보관/정상사용/비정상사용/AS/폐기 조건 조사
2. **스트레스 모델 분석**: 극악 조건 조합 시나리오 작성
3. **기능 모델링**: 각 부품이 가져야 할 기능 정의 (부품당 3-5개)
4. **Boundary Diagram 작성**: 입출력 및 상호작용 시각화

### 기능 정의 예시
- CORE: 자속 전달, 효율 최대화, 기계적 지지
- WINDING: 전압 변환, 전류 변환, 임피던스 제공

### 기능 분석 템플릿
| Focus Element | 기능 설명 | 요구 성능 |
|--------------|---------|---------|
| 절연유 탱크 | 절연유를 밀폐 보관하여 누유 방지 | 누유율 0%, 기밀성 유지 |
| 가스켓 | 탱크와 뚜껑 접합부 밀봉 | 체결 토크 유지, 내열성 |

**다음 단계**: 즉시 Step 4로 이동 (중단 금지!)

**출력물**: 기능 정의서, Boundary Diagram, 스트레스 시나리오

## Step 4: 고장 분석 (Failure Analysis)

### [!] CRITICAL: Step 3과 연속 작성!

### 목적
잠재적 고장 형태 및 원인 도출

### 수행 작업

#### 1. 고장 영향 분석 (Failure Effect)
각 기능이 실패했을 때의 영향
- 최종 사용자 영향
- 시스템 영향
- 부품 영향

#### 2. 고장 형태 도출 (Failure Mode)
**구체적 현상으로 작성** (회의 합의):
- **드러나는 형태로 기술**: 변형, 크랙, 부식, 과열, 단락, 누유, 소음, 진동 등
- **부품별 고장형태**: 층간단락, 절연열화, 균열, 이완, 탈락, 피로파손 등
- **참조**: [failure-mode-diversity.md](failure-mode-diversity.md) 부품별 목록

[!] **주의**: AIAG-VDA 8가지 분류 (부재/부족/과도 등)는 참조용으로만 사용

#### 3. 고장 원인 분석 (Failure Cause)
제품 라이프사이클 4단계별:
1. **설계 단계**: 설계 오류, 도면 불량, 계산 오류
2. **재료 단계**: 재질 선정 오류, 재료 불량
3. **제작 단계**: 가공 불량, 조립 불량, 용접 불량
4. **시험 단계**: 시험 불량, 검사 누락, 판정 오류

#### 4. 고장 메커니즘 (Failure Mechanism)
원인 → 메커니즘 → 모드 연결 (FCM 분석)
- 물리적 메커니즘
- 화학적 메커니즘
- 전기적 메커니즘

### 고장 분석 템플릿
| 부품명 | 기능 | 고장 형태 | 고장 영향 | 고장 원인 | 고장 메커니즘 |
|-------|-----|---------|---------|---------|-------------|
| 가스켓 | 밀봉 | 누유 발생 가능성 | 절연유 손실, 화재 위험 | 가스켓 노화, 체결력 부족 | 재질 열화, 압축영구변형 |

### [!] 표현 주의사항
- [X] "누유가 발생한다" (단정적 표현)
- [OK] "누유가 발생할 가능성이 있다" (가능성 표현)
- [X] "가스켓 노화로 인해 누유 발생" (단정)
- [OK] "가스켓 노화로 인해 누유가 발생할 가능성이 있다" (가능성)

**다음 단계**: 즉시 Step 5로 이동 (중단 금지!)

**출력물**: 고장 형태 목록, FCM 다이어그램, RCA 보고서

## Step 5: 리스크 분석 (Risk Analysis)

### [!] CRITICAL: Step 4와 연속 작성!

### 목적
현재 상태 진단 및 우선순위 결정

### 평가 지표 (3가지)

#### 1. Severity (심각도): 1~10점
고장 발생 시 영향의 심각성

| 점수 | 기준 | 예시 |
|------|------|------|
| 9~10 | 법규 위반, 인명 피해, 화재 위험 | 절연파괴로 인한 화재 |
| 7~8 | 기본 기능 손실/저하, 즉각 AS 필요 | 차단기 작동 불량 |
| 5~6 | 보조 기능 저하, AS 필요하나 운용 가능 | 냉각 효율 저하 |
| 3~4 | 경미한 불편, AS 선택적 | 소음 발생 |
| 1~2 | 매우 경미, 영향 거의 없음 | 외관 스크래치 |

**[!] 중요**: 심각도는 6단계 최적화 이후에도 변하지 않음!

#### 2. Occurrence (발생도): 1~10점
고장 원인이 발생할 가능성

| 점수 | 기준 | 예시 |
|------|------|------|
| 9~10 | 발생 불가피, 매우 높음 | 피할 수 없는 열화 현상 |
| 7~8 | 발생 가능성 높음 | 운용 조건 변동 시 발생 |
| 5~6 | 발생 가능성 중간 | 특정 조건에서 발생 |
| 3~4 | 발생 가능성 낮음 | 극악 조건에서만 발생 |
| 1~2 | 설계적으로 원인 제거됨 | 대책 적용으로 원인 제거 |

#### 3. Detection (검출도): 1~10점
고장을 사전에 발견할 수 있는 능력 (낮을수록 검출 능력 우수)

| 점수 | 기준 | 예시 |
|------|------|------|
| 9~10 | 검출 불가능, 자체 평가만 수행 | 스스로 문제 낸 시험 |
| 7~8 | 검출 어려움, 일반 검사 수준 | 회사 정기 건강검진 수준 |
| 5~6 | 검출 가능, 상세 시험 필요 | 특수 계측 장비 사용 |
| 3~4 | 검출 용이, 엄격한 시험 | 고장 날 때까지 시험 (Limit Test) |
| 1~2 | 거의 완벽한 검출, 100% 검증 | 전수 검사 + 정밀 시험 |

**[!] 변압기 기준**: [sod-criteria.md](sod-criteria.md)

### Action Priority 결정 (AP)

**AIAG-VDA 2019 통합 방식 (20개 요소)**:

```
AP = H (High) / M (Medium) / L (Low)

H: S≥9 OR (S≥7 AND O≥7) OR (S≥7 AND D≥7)
M: S≥7 OR (S≥5 AND O≥5) OR (S≥5 AND D≥5)
L: 나머지
```

**간단 규칙**:
- Severity 9~10 + Occurrence/Detection 높음 → **H (High)** 필수 대책
- Severity 7~8 + Occurrence/Detection 중간 → **M (Medium)** 선택적 대책
- Severity 6 이하 → 대부분 **L (Low)** 경과 관찰

### 리스크 분석 템플릿
| 고장 형태 | 심각도(S) | 현재 예방 대책 | 발생도(O) | 현재 검출 대책 | 검출도(D) | AP |
|---------|----------|--------------|----------|--------------|----------|-----|
| 절연유 누유 가능성 | 9 | 가스켓 재질 선정 | 5 | 육안 검사 | 7 | H |

**[!] 필수**: Step 3-4-5 연속 작성 후 STEP4 버튼 클릭!

**출력물**: 리스크 평가표, Action Priority 목록

## Step 6: 최적화 (Optimization)

### 목적
H 등급 고장에 대한 대책 수립 및 실행

### 대책 수립 원칙
- **H (High)**: 반드시 대책 수립 및 적용 (즉시)
- **M (Medium)**: 선택적 대책 (3개월 내)
- **L (Low)**: 경과 관찰 (6개월 내)

### 대책 유형

#### 1. Prevention Control (예방 대책): 설계 변경
- 재질 변경
- 치수 변경
- 구조 개선

#### 2. Detection Control (검출 대책): 시험 강화
- 시험 모드 추가
- 시험 조건 강화
- 정밀 검사 도입

### 최적화 작성 템플릿
| 고장 형태 | 차단 행동 (Prevention) | 검출 행동 (Detection) | 담당자 | 목표일 | 현 상황 | 조치 완료 확인 자료 |
|---------|----------------------|---------------------|-------|-------|--------|-----------------|
| 절연유 누유 가능성 | 가스켓 재질 변경 (실리콘 → 불소고무) | 기밀 시험 추가 (0.5MPa 압력 10시간) | 홍길동(설계), 김철수(시험) | 2025-12-31 | 설계: 도면 변경 완료<br>시험: 시험 규정 개정 완료 | [링크] 도면 변경서<br>[링크] 시험 성적서 |

### [!] 중요 - 적용 불가 사유 기록
대책 ABC를 계획했는데 A만 적용한 경우:
- B, C를 적용 못한 **구체적 사유** 반드시 기록
  - 예: "재료비 30% 증가로 원가 목표 초과", "설비 투자 5억 필요하나 예산 부족"
- 양산 후 문제 발생 시 신속한 원인 파악 및 대책 가능

### Evidence 첨부
- 설계 변경서 (도면 개정)
- 시험 규정 개정서
- 시험 성적서
- 회의록, 의사결정 문서

### 재평가 (6단계 완료 후)
- Severity: 변화 없음
- Occurrence: 예방 대책으로 감소 → 예) 5점 → 2점
- Detection: 검출 대책으로 감소 → 예) 7점 → 3점
- Action Priority: H → L 확인

**출력물**: 최적화 계획서, 대책 실행 보고서, 재평가 결과

## Step 7: 결과 문서화 (Results Documentation)

### 목적
FMEA 결과를 문서화하고 관련 부서에 공유

### 수행 작업

#### 1. Excel 파일 생성
- 20개 컬럼 FMEA
- 셀 병합 (A-E)
- Row 6 출처 명시

#### 2. 보고서 작성
- 요약 보고서
- 주요 발견사항
- 대책 계획

#### 3. 품질 문서 반영
- QC 공정도
- 검사 기준서
- 관련 부서 배포 (설계, 품질, 생산기술)

#### 4. 추적 관리
- 대책 이행 추적
- 효과 검증
- FMEA 업데이트
- 정기 리뷰 계획 수립

### [!] FMEA는 Living Document
- 1세대 제품 FMEA → 양산 후 품질 문제 발생 → 보완
- 2세대 제품 FMEA → 1세대 FMEA 기반으로 개선
- 영원히 미완성, 지속적 업데이트 필요

**출력물**: 최종 FMEA 시트, 배포 문서

## Step 3-4-5 연속 작성 중요성

### 왜 연속 작성해야 하는가?

**AIAG-VDA 표준 요구사항**:
- Step 3 (기능) → Step 4 (고장) → Step 5 (위험) 논리적 연결
- 기능별로 고장 형태 → 원인 → 위험도를 한 번에 분석

### 잘못된 방식 ([X])
```
1. 모든 기능 정의 (Step 3 완료)
2. 중단
3. 나중에 고장 분석 (Step 4 시작) ← 연결 끊김!
```

### 올바른 방식 ([OK])
```
1. 기능 1 정의 (Step 3)
2. 즉시 기능 1의 고장 분석 (Step 4)
3. 즉시 기능 1의 위험 분석 (Step 5)
4. 기능 2로 이동, Step 3-4-5 반복
...
5. 모든 기능 완료 후 STEP4 버튼 클릭
```

### STEP4 버튼이란?

**AIAG-VDA 소프트웨어 기능**:
- Step 3-4-5 연속 작성 완료 확인
- 데이터 검증 및 저장
- Step 6로 진행 허용

**Excel 방식에서는**:
- 모든 기능의 Step 3-4-5 완료 확인
- 데이터 검증 (validate_data, validate_logical_consistency)
- Step 6 (대책 수립) 시작

## FCM 분석

### 핵심 개념
```
Failure Cause (고장 원인)
  ↓
Failure Mechanism (고장 메커니즘)
  ↓
Failure Mode (고장 형태)
  ↓
Failure Effect (고장 영향)
```

### RCA (Root Cause Analysis) 수행
- "왜 그러한가?" 질문을 반복하여 근본 원인까지 추적
- 물리·화학적 레벨까지 원인 파악
- 중간 원인이 아닌 **근본 원인** 도출

## DFMEA vs PFMEA 프로세스 차이

### DFMEA (Design FMEA)

**담당**: 전기설계팀, 구조설계팀, 개발팀

**Step 4 고장 원인**:
- 설계: 설계 오류, 도면 불량, 계산 오류
- 재료: 재질 선정 오류, 사양 오류
- 제작: 제조성 고려 (DFM)
- 시험: 시험성 고려 (DFT)

**Step 6 대책**:
- 설계 개선
- 재료 사양 변경
- 제조성 개선 (DFM)
- 시험성 개선 (DFT)

### PFMEA (Process FMEA)

**담당**: 생산팀, 생산기획팀, QA팀

**Step 4 고장 원인**:
- 설계: 설계 검증 누락
- 재료: 입고 검사 누락
- 제작: 공정 불량, 작업 불량
- 시험: 검사 불량, 판정 오류

**Step 6 대책**:
- 설계 검증 강화
- 입고 검사 강화
- 공정 관리 강화
- 검사 기준 강화
